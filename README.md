# oss-template-2025

拓殖大学工学部 2025年度『オープンソースソフトウェア』第7回実習用のひな形リポジトリです。

## このリポジトリについて

このリポジトリは、学生がGitHub上でのOSS開発の基本的なフローを学ぶための実習用テンプレートです。

### 学習内容

- **Fork**：他人のリポジトリを自分のGitHubアカウントにコピーする
- **Clone**：自分のGitHubからローカルPCにダウンロードする
- **Branch**：作業用のブランチを作成する
- **Commit**：変更を記録する
- **Push**：ローカルの変更をGitHubにアップロードする
- **Pull Request**：元のリポジトリに変更を提案する（次回の講義で実施）

## ディレクトリ構造

```
oss-template-2025/
├── README.md           # このファイル
├── LICENSE             # ライセンスファイル
└── keywords/           # 学生がキーワード解説を追加するディレクトリ
    └── README.md       # キーワードリスト（自動生成）
```

## 学生の皆さんへ

### 実習の流れ

1. このリポジトリを**Fork**して、自分のGitHubアカウントにコピー
2. Forkしたリポジトリを**Clone**して、自分のPCにダウンロード
3. 新しい**ブランチ**を作成（`add-my-keyword`）
4. `keywords/` ディレクトリに自分の学籍番号のファイル（例：`B1234567.md`）を作成
5. 講義で印象に残ったキーワードとその解説を記入
6. 変更を**コミット**して、自分のGitHubに**Push**
7. Pull Requestを作成（次回の講義で実施）

### ファイルの形式

`keywords/[あなたの学籍番号].md` というファイルを作成し、以下の形式で記入してください：

```markdown
# [選んだキーワード]

## 説明

[キーワードの解説を100〜300字程度で記入]

## 作成者

- 氏名: [あなたの氏名]
- 学籍番号: [あなたの学籍番号]
```

### 例

```markdown
# オープンソース

## 説明

オープンソースとは，ソフトウェアのソースコードが公開されており，
誰でも自由に利用，修正，再配布できるソフトウェアのことです．

このオープンソースの理念により，世界中の開発者が協力して
より良いソフトウェアを作り上げることができます．

代表的なオープンソースソフトウェアとしては，
Linux，Git，Pythonなどがあります．

## 作成者

- 氏名: 山田太郎
- 学籍番号: B1234567
```

## 教員の方へ

### キーワードリストの自動生成

学生のPull Requestを全てマージした後、以下のスクリプトで `keywords/README.md` を自動生成できます。

```bash
# リポジトリのルートディレクトリで実行
./generate_keyword_list.sh
```

このスクリプトは：

1. `keywords/` 内の全 `.md` ファイルをスキャン
2. 各ファイルからタイトル、氏名、学籍番号を抽出
3. 学籍番号順にソートしてリスト化
4. `keywords/README.md` を自動生成

### マージの順序

各学生のファイル名（学籍番号）は一意なので、Pull Requestのマージ順序による競合は発生しません。
安心してどの順序でもマージできます。
