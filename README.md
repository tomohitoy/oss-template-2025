# oss-template-2025

拓殖大学工学部 2025年度『オープンソースソフトウェア』実習用のひな形リポジトリです。

## このリポジトリについて

このリポジトリは、学生がGitHub上でのOSS開発の基本的なフローを学ぶための実習用テンプレートです。

### 学習内容

- **Fork**：他人のリポジトリを自分のGitHubアカウントにコピーする
- **Clone**：自分のGitHubからローカルPCにダウンロードする
- **Branch**：作業用のブランチを作成する
- **Commit**：変更を記録する
- **Push**：ローカルの変更をGitHubにアップロードする
- **Pull Request**：元のリポジトリに変更を提案する（次回の講義で実施）

## ディレクトリ構造

```
oss-template-2025/
├── README.md           # このファイル
├── LICENSE             # ライセンスファイル
├── keywords/           # 学生がキーワード解説を追加するディレクトリ
│   └── README.md       # キーワードリスト（自動生成）
└── scripts/            # 管理用スクリプト
    └── generate_keyword_list.sh  # キーワードリスト自動生成スクリプト
```

## 第7回課題：キーワード解説の作成

### 手順

1. **キーワードの選択**
   - 講義資料で提供される共有エクセルファイルにアクセスする
   - キーワードリストから担当したいキーワードを選び、担当者列に自分の名前を記入する
   - エクセルに記載されているファイル名を確認する

2. **ブランチの作成**
   - このリポジトリをForkする
   - 自分のリポジトリ（origin）に所定の名前のブランチを作成する

3. **ファイルの作成**
   - `keywords/[指定されたファイル名].md` というファイルを作成する
   - 以下の形式で記入する：

```markdown
# [選んだキーワード]

## 説明

[キーワードの解説を200文字以上で記入]

## 参考文献

- [参考にしたサイトのタイトル](URL)
- [参考にしたサイトのタイトル](URL)

## 作成者

- 氏名: [あなたの氏名]
- 学籍番号: [あなたの学籍番号]
- 作成日: YYYY-MM-DD
- 最終更新日: YYYY-MM-DD
```

4. **コミット・プッシュ**
   - 作成したファイルをコミットする
   - 作成したブランチにプッシュする

5. **Pull Requestの作成**
   - 元のリポジトリに対してPull Requestを作成する

### 記入例

```markdown
# オープンソース

## 説明

オープンソースとは，ソフトウェアのソースコードが一般に公開されており，
誰でも自由に利用，修正，再配布できるライセンス形態のことです．

このオープンソースの理念により，世界中の開発者が協力して
より良いソフトウェアを作り上げることができます．
また，コードが公開されているため，セキュリティ上の問題も発見されやすく，
多くの人の目によってソフトウェアの品質が向上していきます．

代表的なオープンソースソフトウェアとしては，
Linux，Git，Python，Apache，MySQLなどがあり，
現代のインターネットやソフトウェア開発において不可欠な存在となっています．
```

### 評価基準（ルーブリック）

この課題は以下の3つの観点で評価されます（4段階評価、3が期待通りのパフォーマンス）：

#### 1. 実習の手続き（できた/できなかった）

- ✅ **クリア**：Forkの後、自分のoriginに所定の名前のブランチを作成し、そのブランチに作成したキーワードの解説ファイルをコミット・プッシュできている
- ❌ **未達成**：上記の手順が完了していない

#### 2. キーワードの数

- キーワードを1つ作成（最低限の評価）
- キーワードを2つ作成（より高い評価）

#### 3. 解説の内容

- **1**：100文字未満の解説
- **2**：100文字以上の解説
- **3**：200文字以上の解説
- **4**：200文字以上の解説 + 自分で調べた内容とその引用リスト（リンク）を含む
