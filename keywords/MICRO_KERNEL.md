# [マイクロカーネル]

## 説明
マイクロカーネル（microkernel）とは，OS（オペレーティングシステム）の中心部分である「カーネル」に必要最小限の機能だけを残し，その他の機能をユーザ空間へ切り離して実行する構造を採用したOSアーキテクチャのことです。
従来のモノリシックカーネルでは，プロセス管理・メモリ管理・ファイルシステム・デバイスドライバなど多くのサービスがカーネル空間に置かれていました。その結果，カーネルが巨大化し，バグの影響範囲が大きくなったり，安定性や保守性が低下する問題がありました。
これに対しマイクロカーネルでは，カーネルに残すのは プロセス（タスク）管理，メモリ管理の最小限の仕組み，プロセス間通信（IPC）など本当に必須な機能のみ とします。それ以外のファイルシステム，ネットワークスタック，デバイスドライバなどはユーザ空間でサーバプロセスとして動作します。
この構造によって，1つのサーバが故障してもカーネル全体が落ちることを防げるため 信頼性・安全性が高い という利点があります。また，機能を個別に追加・交換しやすく 拡張性や保守性にも優れる とされます。一方で，ユーザ空間との通信（IPC）が多くなるため，性能面ではモノリシックカーネルより不利になる場合があります。
代表的なマイクロカーネルとしては Mach，L4，MINIX 3 などがあり，現代では性能面の工夫が進み，組み込み機器などで広く利用されています。
