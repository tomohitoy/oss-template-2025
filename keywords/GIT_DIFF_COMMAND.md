# git diff コマンド

## 説明
git diff コマンドは2つの入力データ・セットを取得し、それらの間の変更を出力する機能です。
言い換えると、リポジトリ内の2つの状態の間で(作業ツリー、コミットなど)ファイルの変更内容(差分)の確認をすることができます。
対象となるデータ・ソースはコミット・ブランチ・ファイルなどがあります。

git diffにはオプション機能が存在します。
以下にオプションの一部を紹介します。

git diff --color-words
変更を強調表示することができます。

git diff HEAD..リモート名/ブランチ名
git pullをする前にローカルの最新コミットとpull先のリモートレポジトリとの変更点を見ることができます。

git diff リモート名/ブランチ名..HEAD
git pushをする前にリモートレポジトリとこれからpushしたい箇所を表示したい場合に使うことができます、

git diff
git addをする前に変更した箇所とインデックスとの変更点を見ることができます。

git diff --cached
(-cahed)を後ろにつけることで、インデックスと最新のコミットとの変更点を見ることができます。

git diff HEAD^
git commitをした後にコミットした箇所を表示することができます。

git diff ブランチA..ブランチB
ブランチ同士を比較して変更点を見ることができます。

git diff -- 対象のファイルパス
git addをする前に変更したファイルの変更点だけをみることができます。

git diff -- ファイルパスA ファイルパスB
別ファイル同士を比較することもできます。

git diff --stat
どれくらい変更したかを見ることができます。

git diff --name-only
変更があったファイル名だけを表示することができます。

git diff -w
(-w)をつけることで改行コードの違いや文頭・文末の空白を無視することができます。

git diff --ignore-blank-lines
空行を無視することができます。
